# Hermes Agent 安装与配置完成

## 任务目标
用户要求安装 Hermes Agent v0.10.0 Web UI 版本，并使用 QClaw 的 token。

## 执行过程

### 1. 安装 Hermes Agent
- 使用官方中国镜像安装脚本: `irm https://res1.hermesagent.org.cn/install.ps1 | iex`
- 初次安装因虚拟环境文件被锁定失败
- 解决：停止相关 Python 进程后重新安装
- 版本: Hermes Agent v0.10.0 (2026.4.16)
- Python: 3.12.10

### 2. OpenClaw 迁移
- 执行: `hermes claw migrate --yes`
- 迁移结果: 2 个技能已迁移
  - find-skills
  - skillhub-preference

### 3. 配置 QClaw Gateway 集成
发现 QClaw 提供本地 LLM 代理端点:
- `QCLAW_LLM_BASE_URL = http://127.0.0.1:19000/proxy/llm`
- `QCLAW_LLM_API_KEY = __QCLAW_AUTH_GATEWAY_MANAGED__`

配置 Hermes 使用此端点:
```
hermes config set model.provider custom
hermes config set model.base_url http://127.0.0.1:19000/proxy/llm
hermes config set model.default qclaw/modelroute
```

## 使用方法
```powershell
# 重启终端让 PATH 生效后
hermes          # 启动聊天
hermes config   # 查看配置
hermes setup    # 重新配置
```

## 注意事项
1. QClaw Gateway 必须在 19000 端口运行
2. API Key 由 QClaw Gateway 自动管理，无需手动配置
3. 如需使用其他模型提供商，可通过 `hermes config set` 修改
