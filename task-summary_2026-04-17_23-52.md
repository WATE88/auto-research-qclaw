## 问题
用户询问 GitHub 研究和本地研究有什么区别

## 发现

### 1. 代码差异
- **唯一修改文件**：`autoresearch_autorun.py`
- **修改内容**：并发控制修复（status="pending" → status="running"）
- **目的**：防止竞态条件导致任务重复启动

### 2. 数据差异
| 文件类型 | GitHub | 本地 |
|---------|--------|------|
| 进化数据库 | ❌ 无 | `evolution_monitor.db` |
| 任务数据库 | ❌ 无 | `autorun_monitor.db` |
| 运行日志 | ❌ 无 | `evolve_start.log`, `notify.log` |

### 3. 运行效果对比
- **GitHub 版本**：Gen 2-3 崩溃，并发任务失控
- **本地修复版**：Gen 0-8+ 稳定运行，并发 ≤2

## 结论
本地版本更稳定。建议继续验证后再 push 到 GitHub。
