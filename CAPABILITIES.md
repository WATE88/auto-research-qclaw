# AutoResearch v3.2 完整能力清单

## 🎯 核心能力

### 1. 多源信息采集
- **ProSearch**: 中文搜索引擎（实时网页）
- **GitHub**: 项目发现（4次指数退避重试）
- **HackerNews**: 技术讨论社区
- **ArXiv**: 学术论文库（自动关键词优化）
- **Reddit**: 社区讨论（可选）

### 2. 智能关键词优化
- **中文→英文自动翻译**: 42个技术术语
- **TurboQuant 相关**: KV缓存、量化、无损压缩等
- **技术学术激活**: 英文关键词激活 ArXiv（中文返回 0 篇，英文返回 5+ 篇）

### 3. 论文质量评分系统
- **质量信号词**: benchmark, evaluation, lossless, zero precision (+2.0)
- **低质量过滤**: survey, review, preliminary (-0.5 ~ -1.5)
- **摘要长度检查**: <200字 -1.0, >500字 +0.5
- **主题相关性**: 标题与查询词重叠加分

### 4. 论文时效性权重
- **2026年**: ×2.0（最新成果优先）
- **2025年**: ×1.5
- **2024年**: ×1.0
- **更早**: ×0.5

### 5. 聚类去重机制
- **Jaccard 相似度**: >0.5 则认为同类
- **每类保留最高分**: 减少冗余 +35%
- **15篇 → 5篇**: 高质量论文集中

### 6. 多样性量化评分
- **Simpson 多样性指数**: 类型 + 来源双维度
- **范围**: 0.0（单一）~ 1.0（完全均匀）
- **权重**: 多样性 ×10（综合评分）

### 7. 探索-利用平衡策略
- **前 50% 轮次**: 探索新源（尝试 GitHub、ArXiv）
- **后 50% 轮次**: 利用最优源（深挖高贡献源）
- **Karpathy 式闭环**: 有效提升→继续演进，无效→回退

### 8. 智能深度调节
- **多样性 < 0.4**: 升级到 deep（多源深挖）
- **多样性 0.4-0.6**: 保持 standard
- **多样性 > 0.6**: 保持或降低（已足够多样）

### 9. 自适应缓存 TTL
- **热门主题**: 10 分钟（频繁访问）
- **中等主题**: 30 分钟
- **冷门主题**: 2 小时（很少访问）

### 10. 重排序优化
- **novelty × relevance**: 新颖性 ×0.6 + 相关性 ×0.4
- **去重后重排**: 保留最高分结果

### 11. 增强评分函数
```
综合评分 = 信息量×1.0 + 热度×0.01 + 多样性×10
```

### 12. 自主探索模式 ⭐ NEW
- **自动主题生成**: 从搜索结果提取新兴话题
- **迭代研究**: 持续探索相关主题
- **避免重复**: 历史主题去重
- **命令**: `--explore --explore-depth N`
